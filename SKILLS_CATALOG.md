## ⚠️ Notes

- **Doublons** : `seo-local` (claude-seo) vs `ecoride-seo-local` (mien). Le mien est spécifique à Neuilly-sur-Seine, l'autre est générique.
- **`programmatic-seo`** existe en marketing + `seo-programmatic` en claude-seo (similaires).
- **Duplicate sur `seo-seo-audit`** : nom préfixé automatiquement par collision avec un autre, l'original `seo-audit` existe aussi.
- **Lazy loading** : Claude ne charge le contenu d'un skill QUE quand il est invoqué via `/<nom>` ou pertinent au contexte. Avoir 115 skills n'alourdit pas le contexte par défaut.

## Comment invoquer

```
/<nom-du-skill>
```

Ex : `/ecoride-visual-fusion-system`, `/seo-local`, `/owasp-security`, `/systematic-debugging`.

Ou simplement décrire ce qu'on veut : Claude détecte automatiquement le skill pertinent quand la `description:` du frontmatter match l'intention.
