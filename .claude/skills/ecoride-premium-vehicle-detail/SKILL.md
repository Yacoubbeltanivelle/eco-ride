# ECO RIDE Premium Vehicle Detail

## Mission

Create a vehicle detail page that converts.

It must make the visitor think:

> "I understand the vehicle, the price, the conditions and the next step."

## Page structure

Required sections:

1. Immersive vehicle hero
2. Price and availability
3. Key specs
4. Vehicle score
5. Gallery
6. Services included
7. Conditions
8. Documents needed
9. Financing / rental explanation placeholder
10. FAQ
11. Similar vehicles
12. Sticky CTA

## Hero

Hero can be light or dark depending on page context.

Must include:

- large vehicle image;
- brand/model/version;
- status badge;
- location/vente badge;
- price;
- short positioning text.

## Key specs chips

Show:

```
Année
Kilométrage
Énergie
Boîte
Puissance
Places
Portes
Crit'Air
CO₂ if available
```

## Vehicle score

Create a visual score component.

Categories:

- Économie
- Confort
- VTC
- Écologie
- Disponibilité

Use progress bars, radial bars or compact cards.

## Price module

If rental:

- Prix/semaine HT
- Prix/mois HT
- Caution
- Km inclus
- Assurance/entretien/assistance according to formula

If sale:

- Prix TTC
- Garantie placeholder
- Frais à prévoir placeholder
- Financement possible placeholder

Always mark uncertain legal/commercial details as demo placeholders.

## Sticky CTA

Mobile:

- Appeler
- WhatsApp
- Demander

Desktop:

- right sticky panel;
- summary;
- CTA;
- contact options.

## Documents block

For VTC/pro rental, show document checklist:

- Permis
- Pièce d'identité
- Justificatif de domicile
- KBIS or statut pro if applicable
- Carte VTC if applicable
- RIB if applicable

Mark as "liste indicative à confirmer par ECO RIDE".

## Trust block

Show:

- véhicule contrôlé;
- révision;
- accompagnement;
- démarches transparentes;
- disponibilité à confirmer.

## FAQ examples

- Peut-on réserver rapidement ?
- Quels documents fournir ?
- La caution est-elle obligatoire ?
- L'assurance est-elle incluse ?
- Le véhicule est-il compatible VTC ?
- Peut-on acheter après location ?

## Avoid

- Hiding the price.
- Making the CTA hard to find.
- Too much technical information before benefits.
- Fake guarantees.
- Fake legal promises.

## Acceptance criteria

The detail page is good if:

- Price is visible quickly.
- CTA is obvious.
- Specs are scannable.
- Trust is strong.
- Mobile layout feels premium.
